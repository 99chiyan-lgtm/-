# 教學網頁開發工作流程

## 步驟一：裝置選擇（必須先確認）

| 裝置 | 適用情境 | 尺寸 | 注意事項 |
|------|----------|------|----------|
| **手機 9:16** | 社群媒體、Reels、限動 | 1080×1920 | 最常用，字體要大 |
| **平板 4:3** | 居服員上課用iPad | 768×1024 | 版面較寬，可放更多內容 |
| **電腦 16:9** | 線上課程、投影播放 | 1920×1080 | 可放左右對照版面 |

**👉 請選擇：（1 / 2 / 3）**

## 步驟二：頁面滿邊設計（三選一）

### 選項 A：全滿邊（無留白）
```
┌──────────────────┐
│  標題文字內容     │
│  內容文字內容     │
│  按鈕            │
└──────────────────┘
```
- **優點**：畫面飽滿、視覺衝擊力強
- **缺點**：手機邊緣可能被系統UI遮擋
- **適用**：影片、社群貼文

### 選項 B：左右留白（安全邊距）
```
┌──────────────────┐
│  ┌────────────┐  │
│  │ 標題文字   │  │
│  │ 內容文字   │  │
│  │ 按鈕       │  │
│  └────────────┘  │
└──────────────────┘
```
- **優點**：安全、不會被遮擋、閱讀舒適
- **缺點**：畫面較空
- **適用**：教學網頁、表單

### 選項 C：上下留白（內容居中）
```
┌──────────────────┐
│                  │
│  標題文字內容    │
│  內容文字內容    │
│  按鈕            │
│                  │
└──────────────────┘
```
- **優點**：視覺聚焦、專業感
- **缺點**：上下空間可能浪費
- **適用**：登入頁、結尾頁

**👉 請選擇：（A / B / C）**

### 修改規則

1. **改持續秒數**：同步調整字幕最後一個延遲值，確保字幕有時間顯示
2. **改字幕延遲**：最後一個延遲值必須 < 持續秒數 - 2
3. **一次只改一個欄位**：改完測試後再改下一個
4. **修改後標註**：在 git commit 說明中寫「原：XX → 改：YY」

## 步驟四：修改流程

```
1. 告訴我：「我要改第 X 張投影片的 YY」
2. 我先列出：原設定值 → 建議新值
3. 你確認：「好」或「再調整」
4. 我執行修改
5. 等 2 分鐘後重新整理測試
6. 確認 OK → 再改下一個
```

## 步驟五：部署前檢查

- [ ] 所有連結都是相對路徑
- [ ] 沒有敏感檔案（PDF、個資）
- [ ] 手機字體夠大（≥1.1rem）
- [ ] 按鈕夠大（≥44×44px）
- [ ] 背景音樂有播放按鈕
- [ ] 各頁面可互相跳轉
- [ ] 測試至少 3 支手機尺寸
